# Application audit — 2026-07-22

## Scope

The audit covered repository structure, runtime/tooling, dependencies, database access, authentication, federation, API boundaries, client data loading, modal/navigation accessibility, security headers, tests, migrations, and developer guidance.

## Resolved findings

| Priority | Finding | Resolution |
| --- | --- | --- |
| Critical/High | Next.js, Vitest, and transitive Sharp advisories | Updated patched releases, pinned Sharp 0.35.0, regenerated the lockfile, and added a high-severity CI audit gate |
| High | Federation handshake accepted hostnames without DNS/IP SSRF enforcement | Added canonical URL, HTTPS, DNS, resolved-IP, loopback/private/reserved range, path, credential, query, fragment, and redirect controls |
| High | A signed peer could claim another `originInstanceId` | Bound every event origin to the authenticated signing peer |
| High | Public WhatsApp QR creation could be abused for database/CPU load | Added per-IP challenge limits, a bounded unknown-address fallback, and retention cleanup; clients no longer retry a 429 in a loop |
| Medium | Federation replay nonces accumulated indefinitely | Added indexed retention cleanup while preserving the replay window |
| Medium | Production database access could create a new client per call | Reused one PostgreSQL client and Drizzle instance per warm runtime |
| Medium | Federation dispatch repeatedly loaded instance identity | Loads and signs with one identity snapshot per delivery |
| Medium | Remote authors with the same display name could be merged | Added an optional stable origin author id with backward-compatible fallback and a forward migration |
| Medium | Client components duplicated fragile JSON/error parsing | Added typed `fetchJson`/`jsonRequest`, cancellation for initial loads, and regression tests |
| Medium | Modals and the mobile drawer had inconsistent focus/scroll behavior | Added reusable dialog focus trapping, Escape handling, background lock, focus restoration, and inert closed navigation |
| Medium | API and page security headers were incomplete | Added no-store API caching and CSP, HSTS, opener/resource, DNS prefetch, and cross-domain policy headers |
| Low | Generated runtime versions varied by shell | Added `.nvmrc`, explicit pnpm/package engines, CI runtime setup, and troubleshooting guidance |

## Dependency status

`pnpm audit --audit-level high` passes with no high or critical findings.

Two moderate and one low toolchain advisories remain:

- an old esbuild copy inside Drizzle Kit's legacy loader, affecting only esbuild's development server, which this project does not run;
- a newer esbuild advisory affecting its Windows development server, while project development is not served through esbuild;
- Next.js currently pins a PostCSS version below the advisory fix; the described issue requires parsing attacker-controlled CSS and embedding it in a style tag, which this application does not do.

Forcing incompatible transitive versions would create greater migration/build risk. CI blocks any future high or critical finding, and these toolchain paths should be removed when their direct packages publish compatible upgrades.

## Added engineering controls

- Root `AGENTS.md` with product invariants, security rules, runtime commands, database policy, and Definition of Done.
- `PLANS.md` for risky or multi-module work.
- `docs/DEVELOPMENT.md` and `docs/CODE_REVIEW.md`.
- GitHub Actions quality gate for frozen install, lint, TypeScript, tests, build, and dependency audit.
- Regression tests for federation endpoint validation and the shared client API helper.
